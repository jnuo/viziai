### 3.5 URL Structure for Multi-Language SEO

**Recommendation: Subdirectory approach with locale prefix**

```
viziai.app/              → Default (redirect based on browser)
viziai.app/en/           → English
viziai.app/en/blog/      → English blog
viziai.app/de/           → German
viziai.app/de/blog/      → German blog
viziai.app/tr/           → Turkish
viziai.app/tr/blog/      → Turkish blog
viziai.app/es/           → Spanish
viziai.app/es/blog/      → Spanish blog
viziai.app/fr/           → French
viziai.app/fr/blog/      → French blog
```

**Why subdirectories over subdomains:**

- Consolidates domain authority (critical for new domain viziai.app)
- Next.js App Router with next-intl already supports `[locale]` prefix routing
- Easier to manage (one deployment, one sitemap, one robots.txt)
- 75% of hreflang implementations have errors — subdirectories simplify this

**For country-level targeting within a language (e.g., en-AU vs en-GB):**

- Use hreflang tags, NOT separate subdirectories
- `<link rel="alternate" hreflang="en-AU" href="https://viziai.app/en/blog/blood-test-results-australia" />`
- `<link rel="alternate" hreflang="en-GB" href="https://viziai.app/en/blog/blood-test-results-uk" />`

```
Landing Page (viziai.app/en/)
  ├── Blog Hub (/en/blog/)

  │     │     ├── "What does high creatinine mean?" → links to kidney panel metrics

  │     ├── Country Guides ← link to each other + metric explainers
  │     │     ├── "Blood tests in Australia" → links to AU-specific metrics

  │     ├── Comparison Pages ← link to product + country guides
```

### 5.2 Hreflang Tags Implementation

**For Next.js App Router with next-intl:**

```tsx
// In layout.tsx or page-level generateMetadata:
export async function generateMetadata({ params }: Props): Promise<Metadata> {
  const locale = params.locale;
  return {
    alternates: {
      canonical: `https://viziai.app/${locale}/blog/article-slug`,
      languages: {
        en: "https://viziai.app/en/blog/article-slug",
        de: "https://viziai.app/de/blog/article-slug",
        tr: "https://viziai.app/tr/blog/article-slug",
        es: "https://viziai.app/es/blog/article-slug",
        fr: "https://viziai.app/fr/blog/article-slug",
        "x-default": "https://viziai.app/en/blog/article-slug",
      },
    },
  };
}
```

**Critical rules:**

- Every page must have hreflang tags pointing to ALL language versions
- Include `x-default` pointing to the English version
- Hreflang must be bidirectional (page A links to B, B links back to A)
- Render hreflang in server-side HTML, NOT client-side JS (Google may not execute JS for hreflang detection)
- For country-specific pages (e.g., blood-test-results-australia), use `hreflang="en-AU"`

### 5.6 XML Sitemap

Generate dynamically via Next.js `sitemap.ts`:

```typescript
// web/src/app/sitemap.ts
import { locales } from "@/i18n/config";

export default async function sitemap() {
  const blogPosts = await getAllBlogPosts(); // from MDX content layer

  const entries = [];

  for (const locale of locales) {
    // Static pages
    entries.push({
      url: `https://viziai.app/${locale}`,
      lastModified: new Date(),
      changeFrequency: "monthly",
      priority: 1.0,
    });

    // Blog posts
    for (const post of blogPosts.filter((p) => p.locale === locale)) {
      entries.push({
        url: `https://viziai.app/${locale}/blog/${post.slug}`,
        lastModified: post.updatedAt,
        changeFrequency: "monthly",
        priority: 0.8,
      });
    }
  }

  return entries;
}
```
